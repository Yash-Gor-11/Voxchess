# VoxChess ‚Äî Scaffold Build Plan

Single-pass build matching the v2 spec. Public + auth pages and `/play` are fully implemented; remaining authenticated pages render polished, on-brand stub cards inside the real app shell so layout, sidebar, header, theme, and voice bar are all proven.

## Stack adaptations (vs spec)

The spec is already TanStack Start‚Äìoriented. Two small deviations from the prose:
- This template uses **Tailwind v4** with tokens in `src/styles.css` (`@theme inline`, oklch). I'll add VoxChess colors there as semantic tokens (`--brand`, `--accent`, `--accent-chess`, `--board-dark`, `--board-light`) instead of editing a v3 `tailwind.config.ts`.
- No `next-themes` (Next-only). I'll add a tiny `ThemeProvider` (localStorage + `dark` class on `<html>`) with the same `ThemeToggle` UX.
- Auth uses **Lovable Cloud** (Supabase under the hood) ‚Äî I'll enable it during the build.

## Voice UX (centerpiece)

- **NavVoiceButton** ‚Äî blue, 48px, compass icon, `[N]` shortcut, available on every page. Pulses with `animate-mic-blue` while listening.
- **ChessVoiceButton** ‚Äî green, 56px, knight icon, `[Space]` shortcut, only enabled on `/play` and `/analysis/*`.
- Only one mode active at a time; activating one cancels the other (enforced in `voiceStore`).
- Real Web Speech API (`SpeechRecognition` / `webkitSpeechRecognition`) with mock fallback. Firefox / unsupported browsers see a top `Banner` and disabled buttons with explanatory tooltips.
- Global key bindings live in `__root.tsx`; ignored when an `<input>` / `<textarea>` / `contenteditable` is focused. Space gets `preventDefault` only when chess voice activates.
- `VoiceStatusBar` slides in below the header when active, shows pulsing dot (blue/green) + mode label + live transcript, fades out 3s after completion.
- Handlers (`navVoiceHandler`, `chessVoiceHandler`) are stubs that return mock results ‚Äî chess handler runs the parsed move through `chess.js` so spoken moves on `/play` actually update the board.

## /play (full)

3-column layout `[260px | flex-1 | 320px]`:
- **Left**: MoveList (monospace, ply badge) + mock evaluation bar.
- **Center**: status badges (turn / check / game over), `BoardWrapper` (`react-chessboard` with drag-drop, board colors via CSS vars), action row (New / Undo / Save / Export PGN / Resign). New-game dialog asks White/Black/Random.
- **Right**: ChessVoiceButton (large) + NavVoiceButton + "Try saying‚Ä¶" examples + transcript area. GameOverDialog on terminal positions.

Game state via `useChessGame` wrapping a `chess.js` instance; `setGame(new Chess(game.fen()))` on each move to trigger re-render.

## Stub pages

Shared `<StubPage icon title description />` rendered inside `_app` shell ‚Äî full sidebar, header, voice bar all live. Centered card uses semantic tokens (`bg-card/60 backdrop-blur`, `border-border/50`), Lucide icon, "Coming next" badge. Each stub gets a relevant icon (LayoutDashboard, BookmarkCheck, LineChart, Settings, User, Users).

## Auth (Lovable Cloud)

- Enable Cloud at the start of the build.
- `auth.login.tsx` / `auth.signup.tsx`: centered 420px card, logo, email/password (react-hook-form + zod), divider, Google + GitHub OAuth buttons, link to other page.
- `useAuth` hook wraps Cloud's `supabase.auth` (signIn, signUp, signOut, OAuth, session listener).
- `_app.tsx` `beforeLoad` redirects to `/auth/login` if no session.
- AppHeader user menu shows avatar + display name + Sign out.
